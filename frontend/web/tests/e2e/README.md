# E2E-тесты ZIPTrack

Тесты написаны на Playwright и проверяют основные пользовательские сценарии в браузере.

## Что покрыто

- Ролевые ограничения:
  - инженер и снабжение видят склад/каталог только на чтение;
  - складовщик видит вкладку `Движения`;
  - CSV-приемка открывается из `Движения`.
- Workflow заявки:
  - создание заявки через API от лица заказчика;
  - перевод заявки менеджером на согласование;
  - уведомление автора о смене статуса.
- Комментарии:
  - внутренний комментарий подсвечивается сотруднику;
  - заказчик внутренний комментарий не видит.
- Уведомления:
  - исполнитель получает уведомление при назначении.

## Запуск

Сначала должны быть подняты backend-сервисы и frontend:

```powershell
docker compose up -d
npm run dev -- --host 0.0.0.0 --port 5174
```

Затем в `frontend/web`:

```powershell
npm run test:e2e
```

Интерактивный режим:

```powershell
npm run test:e2e:ui
```

## Учетные записи

По умолчанию тесты ожидают такие логины и пароли:

- `admin` / `P@ssw0rd`
- `manager` / `P@ssw0rd`
- `storage` / `P@ssw0rd`
- `customer` / `111111`
- `ingeneer` / `P@ssw0rd`
- `buy` / `P@ssw0rd`

Если в твоей базе другие пользователи, передай их через переменные окружения:

```powershell
$env:E2E_MANAGER_USER="manager"
$env:E2E_MANAGER_PASSWORD="пароль"
$env:E2E_CUSTOMER_USER="customer"
$env:E2E_CUSTOMER_PASSWORD="пароль"
npm run test:e2e
```

Можно также поменять адреса:

```powershell
$env:E2E_BASE_URL="http://localhost:5174"
$env:E2E_AUTH_URL="http://localhost:8001"
```

Если учетная запись не найдена, соответствующий тест будет пропущен.
